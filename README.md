# D-L
## Drones &amp; Lamps is the most powerful application, that helps to change lamps with drones.
Это проект по реализации сервиса связи типа `СЕРВЕР` ⮕ `Дрон-Станция` ⮕ `Дрон`. 

Дрон осущствляет замену LED лампы на специальных уличных фонарных столбах с соотвтетствующим креплением. Каждый фонарный столб образует собой точку `node` в системе, а все вместе они образуют **бинарное дерево**. В этом дереве выполняется поиск несправных столбов, информация о которых передаётся на сервер. Если причина неисправности кроется в лампе, то от сервера отправляется подготоввленный маршрут к дрон-станции, а уже там назначается дрон для замены. Общение между сервером и дроном происходит при помощи WebSockets (Библиотека SignalR на сервере и WebSockets в прошивке esp32).

## Технический стек

### Frontend
![]( https://img.shields.io/badge/Vue.js+-3-green)
![]( https://img.shields.io/badge/axios.js-1.13+-green)
![]( https://img.shields.io/badge/VueYandexMaps-3+-yellow)
![]( https://img.shields.io/badge/vuex-4.1+-blue)

Фронт представляет собой веб сервис для отслеживания состояния всех имещюихся дронов, столбов и привязанных к ним дрон-станий с отслеживанием статусов и состояний.

Для запуска

```bash 
# В директории ./frontend
npm install

npm run dev
```
### Backend
![]( https://img.shields.io/badge/ASP.NetCore+-8-blue)
![]( https://img.shields.io/badge/SignalR-blue)

Бэкэнд это сервер, который является информатором для дрон-станции. На нём обрабатываются маршруты, данные, логируются действия и статусы дронов.

Для запуска
```bash
# В директории ./backend
# docker (нужен установленный Docker на устройстве с сервером)
docker-compose up
```

или 

```bash
dotnet restore

dotnetn run --urls "http://0.0.0.0:5000"
```


### Esp32 сборка
![]( https://img.shields.io/badge/C++-17-blue)
![]( https://img.shields.io/badge/ArduinoJson-6.21+-blue)
![]( https://img.shields.io/badge/WebSockets-2.4-blue)

Представляет собой мозг рабочего дрона (на данный момент только основу для него), который получая данные с сервера, отправляется по проложенному маршруту. Логирует данныеы и отправляет отчёты на сервер.

Для запуска потребуется установленная среда `PlatformIo` в VSCode. Загрузите это расширение в разделе `Extentions`.

Далее в появившейся вкладке **Platformio** выберите опцию **Pick a folder** и найдите дирректорию `esprtos\RTOSforDrone`. 

Для загрузки проекта на esp32 в нижней части экрана нажмите на символ `⮕`. Прошивка загрузится на esp32 при условии, что вы подключили её по `usb` к вашему устройству. После окончания загрузки, при наличии питания, esp32 будет работать.


## Важное замечание

**Данный проект не является финальной версией продукта и будет дорабатываться. На данный момент ввиду условий создания проекта, esp32 становится хостом для Wifi сети, сервер подключается к ней по соответствующему Wifi коннекту Drone_Network и по паролю, котрые указаны в конфиге прошивки esp32. Для того, чтобы изменить/подключиться/изменить ip адрес Wifi сети, измените его в соответствующем параметре в прошивке esp32.**
